## Scope

- 普通首页无高级筛选时，创建时间继续默认 `today`。
- 当请求包含高级筛选参数（`title`、`sourceId`、`publishedStart`、`publishedEnd`）且没有显式创建时间参数（`range`、`start`、`end`）时，创建时间默认使用 `all`。
- 前端应用高级筛选时，如果用户没有手动调整创建时间，不再把隐含的 `range=today` 写入请求。
- 前端显示的创建时间会跟随隐式默认切换：出现高级筛选时显示“不限”，移除最后一个高级筛选时回到“当天”。
- 用户手动调整创建时间后，再应用高级筛选时保留显式创建时间参数。
- 补充请求解析和筛选面板请求构造测试。

## Out of Scope

- 不改变普通首页无筛选时的 `range=today` 默认行为。
- 不修改 feed 查询语义或数据库筛选逻辑。

## Acceptance

- 访问首页且没有高级筛选参数时，创建时间仍默认“当天”。
- URL 或 API 请求带高级筛选但不带 `range/start/end` 时，解析出的创建时间范围为“不限”。
- 前端应用高级筛选且未手动改创建时间时，请求不带隐含 `range=today`。
- 前端显示与请求语义一致，不出现“请求按不限、界面仍显示当天”的错位。
- 用户手动调整创建时间后再应用高级筛选，请求保留显式创建时间范围。

## Execution

- 在请求解析层识别高级筛选和显式创建时间筛选，只有“高级筛选存在且创建时间未显式设置”时才默认 `all`。
- 首页继续以 `today` 作为普通默认值，并把初始 URL 是否有显式创建时间传给 `FeedPanel`。
- `FeedPanel` 跟踪创建时间是否被用户显式调整；构造高级筛选请求时，未显式调整则省略 `range` 并同步显示为“不限”。
- 增加测试覆盖条件默认、隐式 `today` 省略、手动创建时间保留、移除高级筛选后回到隐式今日。

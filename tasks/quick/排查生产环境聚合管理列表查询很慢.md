## Debugging Evidence

| Field | Value |
| --- | --- |
| Repro Steps | 生产容器内用 Prisma/SQLite 复现 `/api/admin/clusters?search=...` 查询 |
| Observed Failure | 默认列表约 42-74ms；搜索 `OpenAI` 约 4.2s，`星际之门` 约 4.6s |
| Expected Behavior | 搜索查询应在当前生产数据规模下接近瞬时返回 |
| Root Cause | 原 Prisma `contains` 会按搜索词生成多字段 `LIKE '%term%'`，并对每个聚合组通过 child `items.some` 相关子查询扫描 `summaryText`、`rssExcerpt`、`rssContent`、source name 等字段；同时还会执行 page 和 count 两条重查询 |
| Fix Hypothesis | 带 search 时先用 CTE 汇总匹配 cluster id，再按 id 拉当前页详情；并按用户要求只保留聚合标题和子条目标题搜索 |
| Regression Validation | 本地集成测试覆盖子条目标题中文模糊搜索、singleton minItemCount、以及摘要/详情不参与搜索 |

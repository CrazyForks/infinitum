## Scope

- 将 `/daily/[date]` 服务端渲染拆成匿名公开视图：SSR 只读取已发布日报，不读取管理员 session，不输出草稿内容。
- 为 `/daily/:date` 增加边缘缓存响应头，并保留 Next `revalidate=300`。
- 管理员访问未发布日报时，客户端先确认管理员 session，再通过受保护的 admin API 拉取草稿详情。
- 补充组件级与 admin API 集成测试，覆盖匿名不拉草稿、管理员可水合草稿详情。

## Out of Scope

- 本轮不调整生产 Nginx 或 Cloudflare 规则；应在新代码部署后再把 `/daily/yyyy-mm-dd` 纳入 CDN/反代缓存路径。
- 不调整投票、PV/UV、继续阅读、RSS/export 的已有策略。
- 不改变日报生成、发布、取消发布、微调保存的业务语义。

## Acceptance

- 匿名访问 `/daily/[date]` 只会拿到已发布日报，并可被 `s-maxage=300, stale-while-revalidate=600` 缓存。
- 匿名访问未发布或不存在日报时不会暴露草稿内容。
- 管理员打开未发布日报详情页时，客户端水合后仍能看到草稿详情和日报微调控件。
- `/api/admin/daily-reports/[date]` 只有管理员可读，并以 admin 可见性返回草稿/已发布详情。

## Execution

- 拆掉日报详情页服务端管理员 session 依赖，固定以匿名公开视图读取 `getDailyReportByDate(date, false)`。
- 允许未发布或缺失日报返回一个 noindex shell，让管理员客户端水合有入口；公开 JSON-LD 仅在已发布详情存在时输出。
- 新增 admin detail GET API，复用 `requireAdmin()` 和 `getDailyReportByDate(date, true)`。
- 在详情组件里复用 `useClientAdminSession`，管理员确认后再拉 admin detail；匿名访客不会请求草稿详情 API。
- 增加 `/daily/:date` `Cache-Control`，并补充水合与 API 测试。

## Assumptions

- `/api/admin/session` 与 `/api/admin/daily-reports/[date]` 在 Nginx 和 Cloudflare 上继续 bypass cache。
- 300 秒 `s-maxage` 与 600 秒 SWR 对日报详情新鲜度可接受。
- 管理员草稿详情可接受客户端水合后短暂出现“日报不存在/正在加载日报”状态。

## Risks

- 匿名未发布/缺失日报从 404 变成 noindex shell 200，SEO 语义有变化；已在 Follow-up 记录替代方案。
- 发布或取消发布后，匿名边缘缓存最多按 `s-maxage/SWR` 策略延迟刷新；这是本轮性能优化的有意取舍。

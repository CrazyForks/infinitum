# Infinitum 首页分组侧边栏与整体宽度对齐 Lumina 设计

日期：2026-04-19

## 背景

`infinitum` 首页当前已经在列表卡片、分页、操作按钮和筛选区上持续向 `Lumina` 靠拢，但还有两个明显差异会直接影响“整体像不像 Lumina”的感受：

1. 分组筛选仍然藏在高级筛选面板中，表现形式是下拉框，不像 `Lumina` 列表页的左侧分类侧边栏。
2. 全局 Header 与首页内容区的版心宽度仍然偏窄，视觉上没有 `Lumina` 列表页那种更开阔的工作区感。

本轮目标是继续以 `/Users/shawn/Documents/GitHub/lumina/frontend/pages/list.tsx` 为直接参考，把首页的“分组筛选入口”和“整体版心宽度”继续收齐。

## 目标

1. 将首页的分组筛选改造成接近 `Lumina` 分类筛选的左侧侧边栏样式。
2. 将首页与后台页的 Header、内容区宽度统一向 `Lumina` 列表页的 `max-w-7xl` 基线靠拢。
3. 保持当前 `groupId` 参数、筛选逻辑、接口调用、分页逻辑不变，只调整 UI 骨架与交互入口。
4. 桌面端优先还原 `Lumina` 布局，移动端保持可用且不增加复杂抽屉交互。

## 非目标

1. 不改动 `/api/feed` 的参数语义与后端筛选逻辑。
2. 不引入新的分组统计接口或新的数据库字段。
3. 不在本轮同步重做来源筛选、排序筛选或高级筛选的全部布局。
4. 不照搬 `Lumina` 的页面级业务逻辑，例如文章数统计、批量操作或移动端筛选抽屉。

## 参考基线

本轮主要参考以下 `Lumina` 位置：

- `frontend/pages/list.tsx`
- `frontend/components/AppHeader.tsx`

重点参考点：

1. 列表页主体采用更宽的 `max-w-7xl` 版心。
2. 列表页桌面端为“左侧分类筛选 + 右侧内容区”的双栏结构。
3. 左侧筛选栏支持收起 / 展开。
4. 当前筛选项使用轻量背景高亮，而不是重型卡片状态。
5. 侧栏本身是白色面板、细边框、轻阴影、可 sticky 的工作区块。

## 现状概览

当前首页骨架位于：

- 页面数据入口：`src/app/page.tsx`
- 列表主体：`src/components/feed/feed-panel.tsx`
- 全局页面壳：`src/components/ui/page-shell.tsx`
- 顶部 Header：`src/components/ui/global-header.tsx`

当前布局问题：

1. `GlobalHeader` 仍使用 `max-w-6xl`，比 `Lumina` 的 `max-w-7xl` 更窄。
2. `PageShell` 在默认模式下使用 `max-w-6xl`，首页没有吃到更宽的工作区宽度。
3. `feed-panel` 虽然支持 `sidebar` 插槽，但首页当前并没有启用分组侧边栏。
4. 分组筛选仍位于高级筛选区域内，和 `Lumina` 分类侧栏是两套完全不同的入口形态。

## 设计结论

本轮采用“直接对齐 Lumina 桌面端列表骨架”的方案：

1. 首页切换为带左侧分组侧栏的双栏布局。
2. 首页、后台页与全局 Header 的版心统一放宽到接近 `Lumina` 列表页。
3. 高级筛选中的“分组”下拉移除，避免与左侧分组侧栏重复。
4. 移动端不做复杂抽屉，分组入口退化为顶部可折叠区块或非侧边栏呈现，但仍使用同一份筛选状态。

这是最接近 `Lumina` 感受且风险最可控的方案；它只改 UI 组织，不改业务筛选链路。

## 布局设计

### 1. 全局 Header 宽度

`src/components/ui/global-header.tsx` 中 Header 内容容器从当前 `max-w-6xl` 放宽到与 `Lumina` 更接近的 `max-w-7xl`。

要求：

- 保留现有 Header 高度、导航项与登录 / 登出交互。
- 只放宽版心，不额外增加新导航内容。
- Header 与首页主体的水平 padding 仍使用当前项目已稳定的 `px-4 / sm:px-6 / lg:px-8` 体系。

### 2. 首页主内容宽度

`PageShell` 的主内容区宽度统一放宽到接近 `Lumina` 列表页的视觉展开度，让首页与后台页共用同一套更宽的工作区基线。

设计要求：

- 首页与后台页主体宽度都与 Header 对齐。
- `PageShell` 继续保留宽度分档能力，但默认工作区宽度要统一放宽。
- 首页、审核页、任务页、管理页都使用同一套更宽版心。
- 不允许出现首页宽、后台窄，或 Header 宽、内容窄的分裂视觉。

### 3. 首页双栏结构

首页主体改成：

1. 左侧：分组筛选侧栏
2. 右侧：现有筛选卡片、状态提示、列表卡片、分页

桌面端布局方向：

- 参考 `Lumina` 的 `flex` 双栏结构
- 左栏宽度接近 `lg:w-56`
- 右栏自适应填充

窄屏方向：

- 小屏幕不强制保留左侧常驻栏
- 分组侧栏改为内容区顶部的可折叠区块或顺序前置区块
- 不新增移动端抽屉、遮罩或多层状态管理

## 分组侧栏设计

### 1. 视觉结构

左侧侧栏参考 `Lumina` 分类筛选样式，包含：

- 面板容器
- 标题行
- 收起 / 展开按钮
- `全部分组`
- 分组列表项

样式要求：

- 白色面板背景
- 细边框
- 轻阴影
- 圆角与当前首页筛选卡一致
- 桌面端支持 `sticky`

### 2. 标题区

标题区显示：

- 左侧图标
- 标题文案：`分组筛选`
- 右侧收起 / 展开按钮

交互要求：

- 桌面端支持收起为窄栏
- 收起时仅保留按钮和最小宽度占位，行为与 `Lumina` 分类侧栏一致
- 展开后显示完整分组列表

### 3. 列表项

分组列表包含：

- `全部分组`
- `availableGroups` 中的全部分组

交互要求：

- 点击即更新当前 `groupId`
- 当前选中项高亮
- 非选中项悬浮时使用轻量背景反馈

文案要求：

- `全部分组` 作为空 `groupId` 的唯一入口
- 当前项目不强制补分组计数；若现有数据层没有稳定计数，侧栏仅显示名称，不额外拼接数量

### 4. 状态联动

分组侧栏与现有筛选链路共用同一份 `groupId` 状态：

- 选中某个分组后，列表重新调用当前已有的 `loadFeed`
- 来源筛选仍基于当前 `groupId` 对 `availableSources` 做过滤
- 分页、标题搜索、排序、时间范围等状态都继续沿用现有逻辑

## 高级筛选调整

高级筛选面板中的“分组”下拉删除。

保留内容：

- 标题模糊搜索
- 信息源筛选
- 创建时间日期范围

原因：

- 分组已有新的主入口
- 双入口会带来状态重复与认知混乱
- `Lumina` 对分类也采用“侧栏主入口 + 其他筛选独立存在”的结构

筛选摘要中的“分组：xxx”保留，因为它属于结果状态说明，不是入口。

## 交互与状态要求

### 1. URL 与筛选参数

以下行为保持不变：

- `groupId` 继续参与 URL 查询参数
- 初次进入页面时，仍可根据 URL 中的 `groupId` 还原筛选状态
- 切换分页、排序、搜索时，`groupId` 不丢失

### 2. 默认行为

默认状态下：

- 侧栏展开
- 当前未选择具体分组时，高亮 `全部分组`

### 3. 移动端

移动端不要求完整复刻 `Lumina` 桌面侧栏，但必须满足：

- 有明确的分组筛选入口
- 不丢筛选能力
- 不出现双入口
- 不因布局收缩导致内容区过窄

- `src/components/feed/feed-panel.tsx`
- `src/components/ui/page-shell.tsx`
- `src/components/ui/global-header.tsx`

文件边界：

- 侧栏样式优先在 `feed-panel.tsx` 内通过局部 class 常量组织。
- 本轮不新增独立侧栏组件文件。

## 测试与验证要求

必须覆盖以下验证点：

1. 首页桌面端存在分组侧栏，且默认展开。
2. 点击 `全部分组` 与具体分组时，会按现有逻辑更新 `groupId` 并触发列表刷新。
3. 高级筛选面板中不再出现“分组”下拉。
4. Header 容器宽度和首页、后台页内容容器宽度已统一放宽到目标版心。
5. 移动端不会因为新增侧栏而出现主内容不可读或筛选入口丢失。

测试优先补在已有首页组件测试文件中，不新建重复测试文件。

## 风险与规避

### 风险 1：全站版心统一放宽后，后台局部模块在更宽容器中显得过散

规避方式：

- 同步检查审核页、任务页、管理页在更宽版心下的卡片排列与留白
- 必要时只收紧页面内部局部区块，不回退全局统一版心

### 风险 2：侧栏与高级筛选状态重复导致逻辑混乱

规避方式：

- 删除高级筛选中的分组下拉
- 只保留一个分组筛选入口

### 风险 3：移动端侧栏复刻过度，造成交互臃肿

规避方式：

- 移动端只保留轻量降级版本
- 不引入抽屉与额外复杂状态机

## 验收标准

完成后应满足：

1. 首页在桌面端一眼看上去就是更接近 `Lumina` 列表页的宽版双栏结构。
2. 分组筛选不再藏在高级筛选中，而是变成左侧主入口。
3. Header 与首页、后台页内容区的宽度都明显比当前版本更宽，并与 `Lumina` 更接近。
4. 现有 `groupId`、来源筛选、标题搜索、分页逻辑不出现回归。
5. 移动端仍可完成分组筛选且界面不失控。

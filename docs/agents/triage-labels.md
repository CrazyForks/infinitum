# Triage Labels

Map Forge Loop's canonical triage roles to the labels or states used by this project.

## Category Roles

| Role | Label / State |
| --- | --- |
| bug | bug |
| enhancement | enhancement |

## State Roles

| Role | Label / State | Meaning |
| --- | --- | --- |
| needs-triage | needs-triage | Maintainer must evaluate the issue. |
| needs-info | needs-info | Reporter or human owner must provide more information. |
| ready-for-agent | ready-for-agent | Fully specified and AFK-ready for an agent. |
| ready-for-human | ready-for-human | Requires human judgment, external access, design choice, or manual QA. |
| wontfix | wontfix | Will not be actioned. Record durable rationale in `.out-of-scope/` when useful. |

## Rules

- Each triaged issue should have one category role and one state role.
- Conflicting state roles block automated triage until a human confirms the intended state.
- Agent-authored triage comments should clearly state they were generated by AI if posted to an external issue tracker.

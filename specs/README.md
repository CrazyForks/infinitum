# Specs

本目录保存需求、设计和契约文档。

| 路径 | 内容 |
| --- | --- |
| `requirements/` | 需求与需求评审 |
| `designs/` | 技术设计 |
| `contracts/` | API、类型、权限、状态和错误码契约 |

需求和设计应优先使用 `CONTEXT.md`、`CONTEXT-MAP.md`、`.ai/project-context.md` 或 `docs/agents/domain.md` 中的领域语言。设计必须标注 `ADR Required: yes/no`；只有难以逆转、没有上下文会令人意外、且存在真实取舍的决策才写入 `docs/adr/`。
